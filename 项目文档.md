# 智慧酒店预订 - 酒店比价平台项目文档

## 项目概述

本项目是一个酒店比价平台的前端实现，参考trivago等酒店比价网站，针对国内酒店数据进行了优化。

## 核心功能

1. **酒店搜索**：目的地输入、日期选择、人数/客房设置、300+城市联想
   - 前端：input输入+城市列表数据
   - 后端：/api/suggestions API，使用includes()方法模糊匹配
   - 数据量：300+主要城市数据

2. **热门酒店推荐**：酒店列表展示、图片、价格、评分
   - 数据源：hotel_info表（98,000条酒店数据）
   - 展示形式：卡片式布局（hotel-card）
   - 信息包含：酒店名、地址、价格、评分、图片

3. **酒店比较**：多选比较功能
   - 前端：checkbox多选
   - 数据获取：hotel_info表数据JOIN provinces表获取省份名称

4. **向导功能**：创意灯泡按钮引导，分步骤解说，自动滚动定位
   - 模块：guide.js
   - UI组件：灯泡图标按钮（guide-items）
   - 功能：分步骤展示平台使用方法

5. **用户认证**：注册/登录/退出，session管理
   - 后端API：/api/register, /api/login, /api/logout, /api/check-login
   - 数据存储：users表（username, password），sessions表（session_id, username）
   - 会话ID：随机生成9位session_id
   - 级联删除：用户删除时自动清理sessions记录

6. **搜索历史**：保存/查看/快速填充，最多10条
   - 存储表：search_history（username, destination, city, hotel, type, check_in, check_out）
   - 搜索类型：ENUM('hotel', 'city')
   - 数据约束：每用户最多保存10条
   - 字段说明：type区分是酒店搜索还是城市搜索

7. **酒店收藏**：收藏/取消收藏/查看收藏
   - 存储表：favorites（hotel_id, hotel_name, hotel_address, hotel_price, hotel_image）
   - 唯一约束：UNIQUE(username, hotel_id)，防止重复收藏
   - 前端模块：favorites.js

### 数据库技术

#### MySQL在项目中的应用

**数据库名称**：hotel_search

**创建的表**：

1. **users表**
   - **结构**：
     - id: INT AUTO_INCREMENT PRIMARY KEY
     - username: VARCHAR(50) UNIQUE NOT NULL
     - password: VARCHAR(50) NOT NULL
     - created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   - **用途**：存储用户账号信息，用于用户认证
   - **数据**：用户注册时插入，包含用户名和密码

2. **sessions表**
   - **结构**：
     - id: INT AUTO_INCREMENT PRIMARY KEY
     - session_id: VARCHAR(50) UNIQUE NOT NULL
     - username: VARCHAR(50) NOT NULL
     - created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
     - FOREIGN KEY (username) REFERENCES users(username) ON DELETE CASCADE
   - **用途**：存储用户会话信息，用于保持用户登录状态
   - **数据**：用户登录时生成session_id并插入，退出登录时删除

3. **search_history表**
   - **结构**：
     - id: INT AUTO_INCREMENT PRIMARY KEY
     - username: VARCHAR(50) NOT NULL
     - destination: VARCHAR(100) NOT NULL
     - city: VARCHAR(50) NULL
     - hotel: VARCHAR(100) NULL
     - type: ENUM('hotel', 'city') NOT NULL DEFAULT 'city'
     - check_in: DATE NOT NULL
     - check_out: DATE NOT NULL
     - timestamp: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
     - FOREIGN KEY (username) REFERENCES users(username) ON DELETE CASCADE
   - **用途**：存储用户搜索历史，支持快速填充和热门搜索统计
   - **数据**：用户每次搜索时插入，包含目的地、城市、酒店、搜索类型、入住和退房日期
   - **特点**：自动限制每条用户最多保存10条历史记录

4. **favorites表**
   - **结构**：
     - id: INT AUTO_INCREMENT PRIMARY KEY
     - username: VARCHAR(50) NOT NULL
     - hotel_id: VARCHAR(50) NOT NULL
     - hotel_name: VARCHAR(100) NOT NULL
     - hotel_address: VARCHAR(200) NOT NULL
     - hotel_price: INT NOT NULL
     - hotel_image: VARCHAR(500) NOT NULL
     - created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
     - UNIQUE KEY unique_user_hotel (username, hotel_id)
     - FOREIGN KEY (username) REFERENCES users(username) ON DELETE CASCADE
   - **用途**：存储用户收藏的酒店信息
   - **数据**：用户收藏酒店时插入，包含酒店ID、名称、地址、价格和图片
   - **特点**：确保每个用户对每个酒店只能收藏一次

5. **hotel_info表**
   - **结构**：
     - HOTELID: VARCHAR(50) PRIMARY KEY
     - CHN_NAME3: VARCHAR(100) NOT NULL
     - PROVINCE: VARCHAR(50) NOT NULL
     - CITY_NAME: VARCHAR(50) NOT NULL
     - CHN_ADDRESS: VARCHAR(200) NOT NULL
     - PET_TEXT: VARCHAR(100) NULL
     - LNG_BAIDU: VARCHAR(50) NULL
     - LAT_BAIDU: VARCHAR(50) NULL
   - **用途**：存储酒店基础信息，用于搜索和展示
   - **数据**：从`sql/酒店信息(1).sql`文件导入，包含酒店ID、名称、省份、城市、地址、宠物政策、百度经纬度等信息
   - **特点**：已导入98,000条酒店数据，覆盖全国主要城市

6. **provinces表**
   - **结构**：
     - id: INT AUTO_INCREMENT PRIMARY KEY
     - province_code: VARCHAR(10) UNIQUE NOT NULL
     - province_name: VARCHAR(50) NOT NULL
     - created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   - **用途**：存储省份编码与中文名称的映射关系
   - **数据**：包含国内34个省份及国外9个地区（香港、台湾、澳门、泰国等），共42条记录
   - **特点**：province_code字段可与hotel_info表的PROVINCE字段关联
   - **关联方式**：
     ```sql
     SELECT h.*, p.province_name 
     FROM hotel_info h
     LEFT JOIN provinces p ON h.PROVINCE = p.province_code
     WHERE h.CITY_NAME = '城市名'
     ```

**MySQL在项目中的具体应用**：
- **用户认证**：通过users表验证用户登录信息
- **会话管理**：通过sessions表保持用户登录状态
- **搜索历史**：通过search_history表记录和管理用户搜索历史
- **热门搜索统计**：通过search_history表统计热门酒店和城市
- **酒店收藏**：通过favorites表管理用户收藏的酒店
- **酒店数据存储**：通过hotel_info表存储酒店基础信息，支持搜索和展示
- **地理位置数据**：通过hotel_info表中的经纬度信息，支持地图展示和距离计算

## API详细说明

### 用户认证相关
| 接口 | 方法 | 请求体 | 响应 | 说明 |
|------|------|--------|------|------|
| /api/register | POST | {username, password} | {success, username, sessionId} | 用户注册，自动创建session |
| /api/login | POST | {username, password} | {success, username, sessionId} | 用户登录，生成session |
| /api/logout | POST | {username} | {success} | 退出登录，删除session |
| /api/check-login | GET | - | {isLogin, username} | 检查登录状态 |

### 搜索相关
| 接口 | 方法 | 参数 | 响应 | 说明 |
|------|------|------|------|------|
| /api/suggestions | GET | query | Array[城市] | 城市名称联想，includes匹配 |
| /api/history | GET | username | Array[搜索记录] | 获取用户搜索历史 |
| /api/history | POST | {username, destination, city, hotel, type, check_in, check_out} | {success} | 保存搜索记录，自动限制10条 |

### 收藏相关
| 接口 | 方法 | 参数 | 响应 | 说明 |
|------|------|------|------|------|
| /api/favorites | GET | username | Array[收藏酒店] | 获取收藏列表 |
| /api/favorites | POST | {username, hotel_id, hotel_name, ...} | {success} | 添加收藏，UNIQUE约束 |
| /api/favorites/:id | DELETE | username | {success} | 取消收藏 |

### 热门相关
| 接口 | 方法 | 参数 | 响应 | 说明 |
|------|------|------|------|------|
| /api/hot-hotels | GET | limit | Array[酒店] | 热门酒店，从hotel_info表查询 |
| /api/hot-cities | GET | limit | Array[城市] | 热门城市，从search_history统计 |

## 技术亮点

### 数据库设计
1. **外键约束与级联删除**
   - 所有涉及username的外键都设置 ON DELETE CASCADE
   - 用户删除时自动清理关联的sessions、search_history、favorites

2. **唯一性约束**
   - provinces表：province_code UNIQUE（省份编码唯一）
   - favorites表：UNIQUE(username, hotel_id)（防止重复收藏）
   - sessions表：session_id UNIQUE（会话ID全局唯一）

3. **搜索类型区分**
   - search_history表使用ENUM('hotel', 'city')字段
   - 前端需判断搜索词是否为酒店关键词：包含"酒店"、"民宿"、"客栈"等则type='hotel'
   - 否则type='city'

4. **地理位置支持**
   - hotel_info表包含LNG_BAIDU和LAT_BAIDU字段
   - 支持后续地图集成和距离计算

### 前端交互优化
1. **日期选择**
   - 使用Flatpickr库，集成airbnb主题
   - 支持中文语言包（l10n/zh.js）
   - 自动禁用过去日期

2. **城市搜索**
   - 实时联想：/api/suggestions API
   - 模糊匹配：includes()方法
   - 300+城市覆盖全国

3. **模块化架构**
   - 各功能独立模块化（js/modules/）
   - 组件化样式（css/components/）
   - 便于维护和扩展

### 国际化支持
- i18next框架自动检测浏览器语言
- 支持中文、英文、美国英语
- 动态切换无需刷新页面
- 包含登录/注册等关键功能的翻译

## 工作流程

### 用户搜索流程
```
1. 输入目的地 → /api/suggestions → 返回城市列表（模糊匹配）
2. 选择日期 → Flatpickr日期选择器
3. 设置人数/客房 → guests.js模块
4. 点击搜索 → 记录search_history（type自动判断）
5. 展示结果 → hotel_info表数据 + provinces表中文名称
```

### 用户认证流程
```
1. 输入用户名/密码 → /api/register或/api/login
2. 服务器验证/创建用户 → users表
3. 生成sessionId → sessions表
4. 保存至localStorage或Cookie
5. 后续请求携带session信息
6. /api/check-login验证登录状态
```

### 酒店收藏流程
```
1. 点击收藏按钮 → favorites.js
2. 发送/api/favorites POST请求
3. 数据库存储酒店信息 → favorites表
4. UNIQUE约束防止重复收藏
5. 获取收藏列表 → /api/favorites GET
6. 取消收藏 → /api/favorites/:id DELETE
```
