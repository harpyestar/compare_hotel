# 干就完了 - 酒店比价平台项目文档

## 项目概述

本项目是一个酒店比价平台的前端实现，参考trivago等酒店比价网站，针对国内酒店数据进行了优化。

## 核心功能

1. **酒店搜索**：目的地输入、日期选择、人数/客房设置、300+城市联想
2. **热门酒店推荐**：酒店列表展示、图片、价格、评分
3. **酒店比较**：多选比较功能
4. **向导功能**：创意灯泡按钮引导，分步骤解说，自动滚动定位
5. **用户认证**：注册/登录/退出，session管理
6. **搜索历史**：保存/查看/快速填充，最多10条
7. **酒店收藏**：收藏/取消收藏/查看收藏

## 技术栈

### 前端技术
- **HTML5**：语义化标签，响应式设计
- **CSS3**：Flexbox, Grid, 动画效果
- **JavaScript (ES6+)**：模块化编程，异步处理
- **Bootstrap 5**：响应式UI框架，组件库
- **Font Awesome**：图标库
- **i18next**：国际化支持
- **Flatpickr**：日期选择器库

### 后端技术
- **Node.js**：JavaScript运行时环境
- **Express**：Web框架
- **MySQL**：关系型数据库
- **CORS**：跨域资源共享

### 数据库技术

#### MySQL
- **定义**：MySQL是一种开源的关系型数据库管理系统(RDBMS)，使用结构化查询语言(SQL)进行数据管理。
- **特点**：
  - 可靠性高，性能稳定
  - 支持事务处理
  - 支持复杂查询
  - 适合处理结构化数据
  - 广泛应用于Web应用开发
- **在项目中的应用**：
  - 存储用户信息
  - 保存搜索历史
  - 管理收藏酒店
  - 记录用户会话

#### Redis (预留)
- **定义**：Redis是一种开源的内存数据结构存储，可用作数据库、缓存和消息代理。
- **特点**：
  - 高性能，基于内存操作
  - 支持多种数据结构（字符串、哈希、列表、集合、有序集合等）
  - 支持持久化
  - 支持发布/订阅模式
  - 适合缓存和会话管理
- **在项目中的潜在应用**：
  - 缓存热点数据（如热门酒店、城市）
  - 存储用户会话
  - 实现分布式锁
  - 计数器功能

## 关键API

| 接口 | 方法 | 功能 |
|------|------|------|
| /api/register | POST | 用户注册 |
| /api/login | POST | 用户登录 |
| /api/logout | POST | 退出登录 |
| /api/check-login | GET | 检查登录状态 |
| /api/history | GET/POST | 搜索历史 |
| /api/favorites | GET/POST | 收藏酒店 |
| /api/favorites/:id | DELETE | 取消收藏 |
| /api/suggestions | GET | 城市联想 |
| /api/hot-hotels | GET | 热门酒店搜索 |
| /api/hot-cities | GET | 热门城市搜索 |
